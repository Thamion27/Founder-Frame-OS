# Risk Sheet

## Risk Categories

| Risk | Description | Mitigation |
|---|---|---|
| Overclaiming | System is marketed as guaranteeing founder success. | Use restrained public language and claims review. |
| Founder dependency | User treats bot as authority instead of mirror/forge/gate. | Repeat: founder is refined, not replaced. |
| Bad advice boundary | User seeks legal, medical, tax, investment, or therapy advice. | Redirect to qualified professionals. |
| Strategic drift | Symbolic language obscures practical execution. | Use primitive extraction and quality gates. |
| Public confusion | Audience thinks this is a belief system or therapy. | Use public-safe language and scope statement. |
