# Founder Frame Scorecard v0.1

Use this scorecard to evaluate a founder idea, product concept, bot prompt, client offer, or public-facing system.

| Dimension | Score 0-5 | What It Measures |
|---|---:|---|
| Vision Clarity |  | Is the vision specific enough to guide decisions? |
| User Specificity |  | Is the intended user/customer clear? |
| Pain Reality |  | Is the problem concrete and important? |
| Mechanism Strength |  | Is the solution mechanism understandable and plausible? |
| Offer Translation |  | Can the idea be expressed as something usable or buyable? |
| Proof Path |  | Is there a realistic way to test demand, value, or effectiveness? |
| Execution Simplicity |  | Can the next step be performed without overbuilding? |
| Quality Discipline |  | Are claims, risks, and assumptions controlled? |
| Originality |  | Does it extract primitives instead of imitating surface forms? |
| Founder Fit |  | Does this align with the founder’s actual strengths and constraints? |

## Interpretation

- **0-15:** Concept fog. Do not build yet.
- **16-30:** Raw potential. Extract primitives and narrow scope.
- **31-40:** Prototype candidate. Run proof test.
- **41-50:** Strong build candidate. Move to sprint architecture.
